# Soc Ops — Workspace Instructions

## 📋 Project Overview
Social Bingo game for mixers (5x5 grid, find people matching questions, 5-in-a-row = win).
**Stack:** TypeScript, React 19, Vite, Tailwind CSS v4 | **Node:** 22+

---

## 🏗️ Architecture

```
src/
├── components/       # React UI (StartScreen, GameScreen, BingoBoard, BingoSquare, BingoModal)
├── hooks/           # useBingoGame (state management)
├── utils/           # bingoLogic.ts (pure functions) + tests (21 passing)
├── data/            # questions.ts (quiz questions)
├── types/           # TypeScript interfaces
└── test/            # bingoLogic.test.ts
```

**Game Flow:** StartScreen → useBingoGame Hook → GameScreen + BingoBoard → BingoModal (win state)

**Core Logic:** `generateBoard()` | `toggleSquare()` | `checkBingo()` | `getWinningSquareIds()`

## 📐 Code Conventions

### React Components
- **Functional only** — no class components
- **Props interface:** `type Props = { ... }`
- **Destructure props** in signature
- **Example:**
  ```typescript
  type Props = { board: Square[]; onSquareClick: (id: number) => void };
  export function BingoBoard({ board, onSquareClick }: Props) { ... }
  ```

### TypeScript
- Strict mode enabled
- No `any` types — explicit types required
- Export types alongside implementations

### Testing (Vitest)
- Unit tests for pure logic (bingoLogic.ts)
- Arrange → Act → Assert pattern
- Descriptive test names

### Styling (Tailwind CSS v4)
- **Only Tailwind utilities** — no inline styles or CSS modules
- CSS variables for theme persistence
- Dark mode support via `dark:` prefix
- Responsive via `sm:` `md:` `lg:` breakpoints
- See: `.github/instructions/tailwind-4.instructions.md`

### Design Philosophy
- **No generic AI aesthetics** (purple gradients, common fonts)
- Create distinctive, memorable UIs
- Typography, color, motion should be cohesive
- See: `.github/instructions/frontend-design.instructions.md`
