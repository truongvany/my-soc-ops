# Soc Ops — Project Instructions

## 📋 Project Overview

**Soc Ops** is a hands-on GitHub Copilot Agent Lab featuring a **Social Bingo game** for in-person mixers. Players find people matching quiz questions and get 5 in a row to win. It demonstrates advanced GitHub Copilot agent workflows and design-first development practices.

- **Status:** Active lab/workshop
- **Stack:** TypeScript, React 19, Vite, Tailwind CSS v4
- **Node.js:** 22+
- **License:** MIT

## 🏗️ Architecture & Project Structure

```
src/
├── components/          # React UI components
│   ├── App.tsx         # Main app router (StartScreen → GameScreen)
│   ├── StartScreen.tsx # Initial UI to begin game
│   ├── GameScreen.tsx  # Main bingo board display
│   ├── BingoBoard.tsx  # 5x5 grid component
│   ├── BingoSquare.tsx # Individual square (question + marked state)
│   └── BingoModal.tsx  # Bingo celebration modal
├── hooks/
│   └── useBingoGame.ts # Game state management, logic orchestration
├── utils/
│   └── bingoLogic.ts   # Pure functions for board generation, validation, scoring
├── data/
│   └── questions.ts    # Quiz questions configuration
├── types/
│   └── index.ts        # TypeScript interfaces & types
└── test/
    └── bingoLogic.test.ts # Full test coverage (21 tests, all passing)
```

## 🎮 Key Concepts

### Game Flow
1. **StartScreen** → User initiates game
2. **useBingoGame Hook** → Manages game state (start, playing, bingo)
3. **GameScreen** → Renders board, handles user clicks
4. **bingoLogic.ts** → Pure functions handle validation
5. **BingoModal** → Celebrates winning state

### Core Logic (`bingoLogic.ts`)
- `generateBoard()`: Creates 25 squares with random question assignments + free center space
- `toggleSquare()`: Mark/unmark squares (immutable updates)
- `checkBingo()`: Detect winning patterns (rows, columns, diagonals)
- `getWinningSquareIds()`: Return highlighted winning squares

**Testing:** Full coverage in `bingoLogic.test.ts` (21 tests, all passing)

## 📐 Code Conventions

### React Components
- **Functional components** only (no class components)
- **Props interface** exported with `type Props = {...}`
- **State via hooks** (useState, useContext, custom hooks)
- **Destructure props** in function signature
- **Forward refs** when needed for imperative access

Example:
```typescript
type Props = {
  board: Square[];
  onSquareClick: (id: number) => void;
};

export function BingoBoard({ board, onSquareClick }: Props) {
  return (
    <div className="grid grid-cols-5 gap-2">
      {board.map((square) => (
        <BingoSquare
          key={square.id}
          square={square}
          onClick={() => onSquareClick(square.id)}
        />
      ))}
    </div>
  );
}
```

### TypeScript
- **Strict mode** enabled in tsconfig
- **Avoid `any`** — use explicit types
- **Export types** alongside implementations
- **Use `const` assertions** for literal types

### Testing with Vitest
- **Unit tests** for pure logic (bingoLogic.ts)
- **Happy path first**, then edge cases
- **Arrange → Act → Assert** pattern
- **Descriptive test names** that read like requirements

Example:
```typescript
it('should detect a complete row', () => {
  const board = generateBoard();
  board[0].marked = true;
  board[1].marked = true;
  board[2].marked = true;
  board[3].marked = true;
  board[4].marked = true; // Complete row

  const result = checkBingo(board);
  expect(result).not.toBeNull();
});
```

### Styling with Tailwind CSS v4
- **CSS variables** for theme consistency
- **Dark mode support** with `dark:` prefix
- **Responsive design** with `sm:`, `md:`, `lg:` breakpoints
- **Motion:** CSS animations via `@keyframes`, avoid over-animation
- **No inline styles** — use utility classes exclusively
- **Review:** [tailwind-4.instructions.md](.github/instructions/tailwind-4.instructions.md)

### Frontend Design
- **No generic AI aesthetics** — create distinctive, memorable UIs
- **Typography:** Choose beautiful, unique fonts (avoid Inter, Arial)
- **Color palettes:** Cohesive aesthetics with dominant colors & sharp accents
- **Motion:** High-impact page load animations, staggered reveals
- **Review:** [frontend-design.instructions.md](.github/instructions/frontend-design.instructions.md)

## 🤝 Working with Copilot

### Tips for Best Results

1. **Use agents, not just Chat**
   - `/tdd` for test-driven development
   - `/pixel-jam` for UI design
   - `/quiz-master` for data/themes

2. **Provide context**
   - Reference existing components: "Like StartScreen.tsx"
   - Link to conventions: "Follow bingoLogic.ts patterns"
   - Specify acceptance criteria in chat

3. **Iterate on design**
   - Ask for multiple aesthetic options
   - Reference color schemes (GitHub Copilot theme, etc.)
   - Request animations/motion details separately

4. **Keep agents scoped**
   - Use `/tdd` for test writing, not UI design
   - Use `/pixel-jam` for UI, not business logic
   - Use `/quiz-master` for quiz content, not component refactor
